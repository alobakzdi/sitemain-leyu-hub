从资料到索引

多数信息管理工具聚焦于存储与展示，而索引效率决定了资料的可复用性。okooo 的思路是将「整理」前置：标签分类、层级嵌套、摘要提取，每个节点都可以作为检索入口。适合知识库、项目文档、笔记集合等场景。支持自定义字段与批量导入，资料规模从几百条到数万条均保持检索响应。

没有过度设计，但保留扩展接口。默认使用 JSON / Markdown 作为数据源，便于版本管理。索引规则可配置，中文分词与拼音模糊匹配均已内建。

索引密度 单条资料可关联 3 级标签、2 组自定义属性、时间戳与原文锚点。筛选响应 < 60ms (10万条以内)。

常见问题 · 摘要

需要什么技术基础才能使用索引功能？
基本文件操作即可。通过网页界面添加资料、打标签、搜索。无需命令行。数据以结构化文本保存。
索引支持哪些文件类型？
直接管理文本、链接、代码片段、短备注。图片与 PDF 可外链引用或存储路径索引。
能否迁移已有数据？
支持从 CSV、JSON、Notion 导出格式进行批量导入，也提供 API 写入接口。
索引之后如何分享或发布？
可生成静态快照，输出为独立 HTML 或部署为索引站点，供团队或公开访问。
